# Шаблон досок на Bootstrap


WebSockets — это технология, которая позволяет открывать сеанс
интерактивной связи между браузером пользователя и сервером. С помощью 
этой технологии пользователь может отправлять сообщения на сервер и 
получать управляемые событиями ответы, не требуя длительного опроса, 
то есть без необходимости постоянно проверять сервер на предмет ответа. 
Подумайте, когда вы отвечаете на электронное письмо в Gmail, и в нижней 
части экрана вы видите всплывающее предупреждение «1 непрочитанное 
сообщение от [...]» от человека, на которого вы только что отвечали. 
Такая обратная связь в режиме реального времени обусловлена такими 
технологиями, как WebSockets!

Источник: [Django: WebSocket`ы и Channels](https://django.fun/articles/tutorials/django-websockety-i-channels/)


REST API (Representational State Transfer API) — это набор архитектурных 
принципов для построения сетевых приложений, использующий стандартные 
HTTP-методы (GET, POST, PUT, DELETE) для взаимодействия между клиентом 
и сервером. Он позволяет двум приложениям обмениваться данными, например, 
когда мобильное приложение запрашивает информацию у веб-сервера, что делает
его простым, масштабируемым и гибким.

Источник: google.com ИИ

## О шаблоне

Этот шаблон на Bootstrap 5 реализует базовую структуру SPA-приложения. Он включает:

- Навигацию: Общая navbar с ссылками на разделы (интегрировать React Router Router).


- Формы авторизации/регистрации: С валидацией полей (клиентская + серверная через JS), ошибками рядом с полями.


- Список досок: Карточки для публичных и личных досок с фильтром, лайками и кнопками доступа.


- Страница доски: Холст фиксированного размера (1600x900) с границами, панелью инструментов, индикатором фокуса и лейблом пользователя.


- Модальные окна: Для создания доски и публичной ссылки.


- Стили: Дополнительные для холста, фокуса, ошибок (подсветка полей).

Интеграция:

- Добавить JS-фреймворк (React/Vue) для динамики, роутинга и состояний.


- Для холста: Интегрировать свою библиотеку вроде Fabric.js для объектов (текст, изображения, фигуры) с поддержкой вращения, масштаба (с aspect ratio для изображений) и фокуса


- API/WebSocket: Использовать `fetch` с `Authorization: Bearer ${token}` для REST, `WebSocket` для реального времени (JSON-данные изменений).


- После регистрации: Перенаправить на `/login`, после логина — на `/public-boards.`


- Публичный доступ: Для `/board/{hash}` не требовать токен, но подключаться к WS для просмотра.
